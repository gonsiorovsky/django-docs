1 """
2 A generic comment-moderation system which allows configuration of
3 moderation options on a per-model basis.
4 
5 To use, do two things:
6 
7 1. Create or import a subclass of ``CommentModerator`` defining the
8  options you want.
9 
10 2. Import ``moderator`` from this module and register one or more
11  models, passing the models and the ``CommentModerator`` options
12  class you want to use.
13 
14 
15 Example
16 -------
17 
18 First, we define a simple model class which might represent entries in
19 a Weblog::
20 
21  from django.db import models
22 
23  class Entry(models.Model):
24  title = models.CharField(maxlength=250)
25  body = models.TextField()
26  pub_date = models.DateField()
27  enable_comments = models.BooleanField()
28 
29 Then we create a ``CommentModerator`` subclass specifying some
30 moderation options::
31 
32  from django.contrib.comments.moderation import CommentModerator, moderator
33 
34  class EntryModerator(CommentModerator):
35  email_notification = True
36  enable_field = 'enable_comments'
37 
38 And finally register it for moderation::
39 
40  moderator.register(Entry, EntryModerator)
41 
42 This sample class would apply two moderation steps to each new
43 comment submitted on an Entry:
44 
45 * If the entry's ``enable_comments`` field is set to ``False``, the
46  comment will be rejected (immediately deleted).
47 
48 * If the comment is successfully posted, an email notification of the
49  comment will be sent to site staff.
50 
51 For a full list of built-in moderation options and other
52 configurability, see the documentation for the ``CommentModerator``
53 class.
54

84 class CommentModerator(object):
85  """
86  Encapsulates comment-moderation options for a given model.
87 
88  This class is not designed to be used directly, since it doesn't
89  enable any of the available moderation options. Instead, subclass
90  it and override attributes to enable different options::
91 
92  ``auto_close_field``
93  If this is set to the name of a ``DateField`` or
94  ``DateTimeField`` on the model for which comments are
95  being moderated, new comments for objects of that model
96  will be disallowed (immediately deleted) when a certain
97  number of days have passed after the date specified in
98  that field. Must be used in conjunction with
99  ``close_after``, which specifies the number of days past
100  which comments should be disallowed. Default value is
101  ``None``.
102 
103  ``auto_moderate_field``
104  Like ``auto_close_field``, but instead of outright
105  deleting new comments when the requisite number of days
106  have elapsed, it will simply set the ``is_public`` field
107  of new comments to ``False`` before saving them. Must be
108  used in conjunction with ``moderate_after``, which
109  specifies the number of days past which comments should be
110  moderated. Default value is ``None``.
111 
112  ``close_after``
113  If ``auto_close_field`` is used, this must specify the
114  number of days past the value of the field specified by
115  ``auto_close_field`` after which new comments for an
116  object should be disallowed. Default value is ``None``.
117 
118  ``email_notification``
119  If ``True``, any new comment on an object of this model
120  which survives moderation will generate an email to site
121  staff. Default value is ``False``.
122 
123  ``enable_field``
124  If this is set to the name of a ``BooleanField`` on the
125  model for which comments are being moderated, new comments
126  on objects of that model will be disallowed (immediately
127  deleted) whenever the value of that field is ``False`` on
128  the object the comment would be attached to. Default value
129  is ``None``.
130 
131  ``moderate_after``
132  If ``auto_moderate_field`` is used, this must specify the number
133  of days past the value of the field specified by
134  ``auto_moderate_field`` after which new comments for an
135  object should be marked non-public. Default value is
136  ``None``.
137 
138  Most common moderation needs can be covered by changing these
139  attributes, but further customization can be obtained by
140  subclassing and overriding the following methods. Each method will
141  be called with three arguments: ``comment``, which is the comment
142  being submitted, ``content_object``, which is the object the
143  comment will be attached to, and ``request``, which is the
144  ``HttpRequest`` in which the comment is being submitted::
145 
146  ``allow``
147  Should return ``True`` if the comment should be allowed to
148  post on the content object, and ``False`` otherwise (in
149  which case the comment will be immediately deleted).
150 
151  ``email``
152  If email notification of the new comment should be sent to
153  site staff or moderators, this method is responsible for
154  sending the email.
155 
156  ``moderate``
157  Should return ``True`` if the comment should be moderated
158  (in which case its ``is_public`` field will be set to
159  ``False`` before saving), and ``False`` otherwise (in
160  which case the ``is_public`` field will not be changed).
161 
162  Subclasses which want to introspect the model for which comments
163  are being moderated can do so through the attribute ``_model``,
164  which will be the model class.
165 
166  """

169  close_after = None

177  def _get_delta(self, now, then):
178  """
179  Internal helper which will return a ``datetime.timedelta``
180  representing the time between ``now`` and ``then``. Assumes
181  ``now`` is a ``datetime.date`` or ``datetime.datetime`` later
182  than ``then``.
183 
184  If ``now`` and ``then`` are not of the same type due to one of
185  them being a ``datetime.date`` and the other being a
186  ``datetime.datetime``, both will be coerced to
187  ``datetime.date`` before calculating the delta.
188 
189  """
190  if now.__class__ is not then.__class__:
191  now = datetime.date(now.year, now.month, now.day)
192  then = datetime.date(then.year, then.month, then.day)
193  if now < then:
194  raise ValueError("Cannot determine moderation rules because date field is set to a value in the future")
195  return now - then

197  def allow(self, comment, content_object, request):
198  """
199  Determine whether a given comment is allowed to be posted on
200  a given object.
201 
202  Return ``True`` if the comment should be allowed, ``False
203  otherwise.
204 
205  """
206  if self.enable_field:
207  if not getattr(content_object, self.enable_field):
208  return False
209  if self.auto_close_field and self.close_after is not None:
210  close_after_date = getattr(content_object, self.auto_close_field)
211  if close_after_date is not None and self._get_delta(timezone.now(), close_after_date).days >= self.close_after:
212  return False
213  return True

248 class Moderator(object):
249  """
250  Handles moderation of a set of models.
251 
252  An instance of this class will maintain a list of one or more
253  models registered for comment moderation, and their associated
254  moderation classes, and apply moderation to all incoming comments.
255 
256  To register a model, obtain an instance of ``Moderator`` (this
257  module exports one as ``moderator``), and call its ``register``
258  method, passing the model class and a moderation class (which
259  should be a subclass of ``CommentModerator``). Note that both of
260  these should be the actual classes, not instances of the classes.
261 
262  To cease moderation for a model, call the ``unregister`` method,
263  passing the model class.
264 
265  For convenience, both ``register`` and ``unregister`` can also
266  accept a list of model classes in place of a single model; this
267  allows easier registration of multiple models with the same
268  ``CommentModerator`` class.
269 
270  The actual moderation is applied in two phases: one prior to
271  saving a new comment, and the other immediately after saving. The
272  pre-save moderation may mark a comment as non-public or mark it to
273  be removed; the post-save moderation may delete a comment which
274  was disallowed (there is currently no way to prevent the comment
275  being saved once before removal) and, if the comment is still
276  around, will send any notification emails the comment generated.
277 
278  """

292  def register(self, model_or_iterable, moderation_class):
293  """
294  Register a model or a list of models for comment moderation,
295  using a particular moderation class.
296 
297  Raise ``AlreadyModerated`` if any of the models are already
298  registered.
299 
300  """
301  if isinstance(model_or_iterable, ModelBase):
302  model_or_iterable = [model_or_iterable]
303  for model in model_or_iterable:
304  if model in self._registry:
305  raise AlreadyModerated("The model '%s' is already being moderated" % model._meta.model_name)
306  self._registry[model] = moderation_class(model)
307 
308  def unregister(self, model_or_iterable):
309  """
310  Remove a model or a list of models from the list of models
311  whose comments will be moderated.
312 
313  Raise ``NotModerated`` if any of the models are not currently
314  registered for moderation.
315 
316  """

321  raise NotModerated("The model '%s' is not currently being moderated" % model._meta.model_name)

325  """
326  Apply any necessary pre-save moderation steps to new

328 
329  """
