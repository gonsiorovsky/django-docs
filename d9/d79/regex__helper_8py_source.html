47  """
48  Given a reg-exp pattern, normalizes it to an iterable of forms that
49  suffice for reverse matching. This does the following:
50 
51  (1) For any repeating sections, keeps the minimum number of occurrences
52  permitted (this means zero for optional groups).
53  (2) If an optional group includes parameters, include one occurrence of
54  that group (along with the zero occurrence case from step (1)).
55  (3) Select the first (essentially an arbitrary) element from any character
56  class. Select an arbitrary character for any unordered class (e.g. '.'
57  or '\w') in the pattern.
58  (5) Ignore comments and any of the reg-exp flags that won't change
59  what we construct ("iLmsu"). "(?x)" is an error, however.
60  (6) Raise an error on all other non-capturing (?...) forms (e.g.
61  look-ahead and look-behind matches) and any disjunctive ('|')
62  constructs.
63 
64  Django's URLs for forward resolving are either all positional arguments or
65  all keyword arguments. That is assumed here, as well. Although reverse
66  resolving can be done using positional args when keyword args are
67  specified, the two cannot be mixed in the same reverse() call.
