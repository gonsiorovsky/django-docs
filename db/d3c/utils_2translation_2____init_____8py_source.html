29 # Here be dragons, so a short explanation of the logic won't hurt:
30 # We are trying to solve two problems: (1) access settings, in particular
31 # settings.USE_I18N, as late as possible, so that modules can be imported
32 # without having to first configure Django, and (2) if some other code creates
33 # a reference to one of these functions, don't break that reference when we
34 # replace the functions with their real counterparts (once we do access the
35 # settings).

37 class Trans(object):
38  """
39  The purpose of this class is to store the actual translation function upon
40  receiving the first call to that function. After this is done, changes to
41  USE_I18N will have no effect to which function is served upon request. If
42  your tests rely on changing USE_I18N, you can delete all the functions
43  from _trans.__dict__.
44 
45  Note that storing the function with setattr will have a noticeable
46  performance effect, as access to the function goes the normal path,
47  instead of using __getattr__.
48  """
