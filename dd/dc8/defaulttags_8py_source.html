897  be displayed by the ``{{ athlete_list|count }}`` variable.
898

900  clauses, as well as an ``{% else %}`` clause that will be displayed if all
901  previous conditions fail. These clauses are optional.
902 
903  ``if`` tags may use ``or``, ``and`` or ``not`` to test a number of
904  variables or to negate a given variable::
905 
906  {% if not athlete_list %}
907  There are no athletes.
908  {% endif %}
909 
910  {% if athlete_list or coach_list %}
911  There are some athletes or some coaches.
912  {% endif %}
913 
914  {% if athlete_list and coach_list %}
915  Both atheletes and coaches are available.
916  {% endif %}
917 
918  {% if not athlete_list or coach_list %}
919  There are no athletes, or there are some coaches.
920  {% endif %}
921 
922  {% if athlete_list and not coach_list %}
923  There are some athletes and absolutely no coaches.
924  {% endif %}
925

927  allowed, for example::
928 
929  {% if articles|length >= 5 %}...{% endif %}
930 
931  Arguments and operators _must_ have a space between them, so
932  ``{% if 1>2 %}`` is not a valid if tag.
933 
934  All supported operators are: ``or``, ``and``, ``in``, ``not in``
935  ``==`` (or ``=``), ``!=``, ``>``, ``>=``, ``<`` and ``<=``.
936 
937  Operator precedence follows Python.
938  """

940  bits = token.split_contents()[1:]

951  conditions_nodelists.append((condition, nodelist))
952  token = parser.next_token()
953 
954  # {% else %} (optional)
955  if token.contents == 'else':
956  nodelist = parser.parse(('endif',))
957  conditions_nodelists.append((None, nodelist))
958  token = parser.next_token()
959 
960  # {% endif %}
961  assert token.contents == 'endif'
962
